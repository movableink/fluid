# Fluid Banner

This Ember component is a shortcut for these styles inherited from the [@movable/tailwind-config - banner component](https://movableink.github.io/tailwind-config/?path=/story/components-banners--all-types)

**Types**
- [Default](#default)
- [Destructive](#destructive)
- [Informative](#informative)
- [Confirmation](#confirmation)
- [Alert](#alert)

**Arguments**
- [Icon _optional_](#icon)
- [Header _optional_](#header)
- [Closing  _optional_](#closing)
- [Yield](#yield)




## Banner Types

See each type for specifics

### Default

This type is rendered when on `type:` argument is passed with `.fluid-banner`

<Story id="components-fluid-banner--default" name="Fluid Banner" height="150px"/>


```hbs
<FluidBanner
  @header="Default Banner"
  @icon="fluid-banner-archive"
  @onClose={{this.mockAction}}
>
  <p>
    Body Content
  </p>
</FluidBanner>
```

## Arugments


### Icon

When passing the icon argument to the component use the full filename so `svg-jar` can render the icon property

<Story id="components-fluid-banner--without-close" name="Fluid Banner" height="150px"/>


```hbs
<FluidBanner
  @header="Default Banner"
  @icon="fluid-banner-archive"
>
  <p>
    Body Content
  </p>
</FluidBanner>
```

### Header

This will be the `h4` rendered to the banner and is Optional. Use this to maintain color contrast ratios important for accessiblity.


### Yield

All elements passed to the block part of the component will be yielded inside of the `.fluid-banner--content` and can be used to style `h4` or `p` with inherited color contrast.

```hbs
<FluidBanner>
  <h4>Default Banner</h4>
  <p>Some banner content</p>
</FluidBanner>
```

Otherwise you'll need to use the proper contrast associated with each type listed here:

https://github.com/movableink/tailwind-config/blob/master/plugins/components/banners.js#L49

  </strong>
